# 音乐解析后端服务 - 保活机制说明

## 概述

后端服务已内置保活机制，确保服务持续稳定运行。

## 保活特性

### 1. 自动异常处理
- **未捕获的异常 (uncaughtException)**：自动捕获并尝试重启服务
- **未处理的Promise拒绝 (unhandledRejection)**：记录日志但不崩溃
- **服务器错误**：捕获并处理端口占用等错误

### 2. 优雅关闭
- **SIGTERM信号**：优雅关闭服务器
- **SIGINT信号**：Ctrl+C时优雅关闭

### 3. 健康检查
- **定期健康检查**：每5分钟记录一次服务器状态
- **健康检查API**：`GET /api/health` 返回服务器状态

### 4. 日志记录
- 所有重要操作都有日志记录
- 错误和警告会被详细记录

## 启动服务

### 开发模式
```bash
npm start
```

### 使用nodemon（自动重启）
```bash
npm run dev
```

## PM2进程管理（可选）

虽然已内置保活机制，但也可以使用PM2进行更高级的进程管理。

### 安装PM2
```bash
npm install
```

### 使用PM2启动
```bash
npm run pm2:start
```

### 其他PM2命令
```bash
# 停止服务
npm run pm2:stop

# 重启服务
npm run pm2:restart

# 查看日志
npm run pm2:logs

# 监控服务
npm run pm2:monit

# 保存进程列表
npm run pm2:save

# 设置开机自启
npm run pm2:startup
```

## 监控建议

### 1. 定期检查日志
查看服务运行状态和可能的错误。

### 2. 使用健康检查
定期调用 `/api/health` 确保服务正常运行。

### 3. 资源监控
监控服务器的CPU和内存使用情况。

## 故障排除

### 服务无法启动
1. 检查端口5000是否被占用
2. 查看错误日志
3. 确保所有依赖已安装

### 服务频繁重启
1. 检查未捕获的异常
2. 查看内存使用情况
3. 增加max_memory_restart限制

## 注意事项

- 内置保活机制已足够应对大多数情况
- 生产环境建议使用PM2或其他进程管理器
- 定期备份重要数据
- 监控服务器资源使用情况
